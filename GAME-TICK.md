# Game Tick Manager Documentation

## Overview

The Game Tick Manager is the core server-side game loop that runs every second to update game state, process queues, handle ship movements, and broadcast changes to connected players.

## Usage

### Basic Setup

```typescript
import { GameTickManager } from '@/game/GameTickManager';
import { Server } from 'socket.io';

// Initialize managers
const planetManager = new PlanetManager();
const shipManager = new ShipManager();
const buildingSystem = new BuildingSystem();
const queueManager = new ConstructionQueueManager();
const movementManager = new ShipMovementManager();

// Create tick manager
const tickManager = new GameTickManager(
  io,
  planetManager,
  shipManager,
  buildingSystem,
  queueManager,
  movementManager,
  {
    tickInterval: 1000,        // 1 second
    saveInterval: 60,          // Save every 60 ticks
    broadcastInterval: 1,      // Broadcast every tick
    enablePerformanceMonitoring: true
  }
);

// Start the game loop
tickManager.start();
```

### Event Handling

```typescript
// Listen to tick events
tickManager.on('started', () => {
  console.log('Game loop started');
});

tickManager.on('tick', (tickCount) => {
  console.log(`Tick #${tickCount}`);
});

tickManager.on('stats', (stats) => {
  console.log('Tick stats:', stats);
});

tickManager.on('stateSaved', ({ duration, planets, ships }) => {
  console.log(`Saved ${planets} planets, ${ships} ships in ${duration}ms`);
});

tickManager.on('error', (error) => {
  console.error('Tick error:', error);
});
```

### Graceful Shutdown

```typescript
const shutdown = async () => {
  console.log('Shutting down...');
  
  // Stop the tick manager (saves final state)
  await tickManager.stop();
  
  // Close other connections
  io.close();
  await db.close();
  
  process.exit(0);
};

process.on('SIGTERM', shutdown);
process.on('SIGINT', shutdown);
```

## Events

### Lifecycle Events

- **`started`** - Game loop started
- **`stopped`** - Game loop stopped
- **`tick`** - Tick completed `(tickCount: number)`

### State Events

- **`stateSaved`** - Game state saved `({ duration, planets, ships })`
- **`saveError`** - Save failed `(error: Error)`

### Performance Events

- **`stats`** - Tick stats collected `(stats: TickStats)`
- **`slowTick`** - Slow tick detected (>500ms)
- **`highMemory`** - High memory usage (>500MB)

### Error Events

- **`error`** - General error `(error: Error)`
- **`tickError`** - Tick processing error `(error: Error)`

## Tick Processing Flow

### 1. Update Planets

```typescript
// For each planet:
for (const planet of planets) {
  // Calculate production from buildings
  const production = buildingSystem.calculateProduction(planet);
  
  // Update resources (per second rate)
  planet.resources.minerals += production.minerals / 3600;
  planet.resources.energy += production.energy / 3600;
  planet.resources.crystal += production.crystal / 3600;
  
  // Track changes
  stateChanges.planets.set(planet.id, planet);
}
```

### 2. Process Building Queues

```typescript
// For each planet with construction queue:
for (const queue of queues) {
  const completed = await queueManager.processQueue(queue.planetId);
  
  if (completed.length > 0) {
    // Add completed buildings to planet
    stateChanges.completedBuildings.push(...completed);
  }
}
```

### 3. Process Ship Queues

```typescript
// For each planet:
for (const planet of planets) {
  const completed = await shipManager.processConstructionQueue(planet.id);
  
  if (completed.length > 0) {
    stateChanges.completedShips.push(...completed);
  }
}
```

### 4. Update Ship Movements

```typescript
// For each moving ship:
for (const ship of movingShips) {
  const arrived = await movementManager.updateShipPosition(ship.id);
  
  if (arrived) {
    stateChanges.arrivedShips.push(ship);
  } else {
    stateChanges.ships.set(ship.id, ship);
  }
}
```

### 5. Broadcast State Changes

```typescript
// For each connected player:
for (const socket of sockets) {
  const userId = socket.userId;
  
  // Filter changes relevant to this player
  const playerChanges = filterPlayerChanges(userId, stateChanges);
  
  // Send only changed data
  if (hasChanges(playerChanges)) {
    socket.emit('planetUpdated', playerChanges.planets);
    socket.emit('shipMoved', playerChanges.ships);
    socket.emit('buildingCompleted', playerChanges.completedBuildings);
    // etc...
  }
}
```

### 6. Periodic Save

```typescript
// Every 60 ticks (60 seconds):
if (tickCount % saveInterval === 0) {
  // Save all planets
  for (const planet of planets) {
    await db.planet.update(planet.id, planet);
  }
  
  // Save all ships
  for (const ship of ships) {
    await db.ship.update(ship.id, ship);
  }
}
```

## Error Handling

### Tick Errors

Errors during tick processing are caught and emitted:

```typescript
try {
  await this.tick();
} catch (error) {
  logger.error('Error in game tick:', error);
  this.emit('tickError', error);
  // Tick continues on next interval
}
```

### Recovery Strategy

1. Log error with context
2. Emit error event for monitoring
3. Continue to next tick (don't crash)
4. Alert if errors persist

### Critical Errors

For critical errors (e.g., database unavailable):

```typescript
tickManager.on('saveError', async (error) => {
  logger.error('Critical: Failed to save game state', error);
  
  // Alert monitoring system
  await alertService.critical('Game state save failed');
  
  // Attempt recovery
  await attemptReconnect();
});
```

### 1. Delta Updates Only

Only broadcast changed data:

```typescript
// BAD: Send all planets every tick
socket.emit('allPlanets', planets);

// GOOD: Send only changed planets
const changedPlanets = stateChanges.planets;
if (changedPlanets.size > 0) {
  socket.emit('planetsUpdated', Array.from(changedPlanets.values()));
}
```

### 2. Batch Operations

Process multiple items together:

```typescript
// Update all planets in transaction
await db.transaction(async (trx) => {
  for (const planet of planets) {
    await trx('planets').where('id', planet.id).update(planet);
  }
});
```

### 3. Async Processing

Use Promise.all for parallel operations:

```typescript
// Process all queues in parallel
await Promise.all(
  queues.map(queue => queueManager.processQueue(queue.planetId))
);
```

### 4. Memory Management

Clear temporary data each tick:

```typescript
// Reset state changes
this.stateChanges = this.createEmptyStateChanges();
```

## Troubleshooting

### Slow Ticks

If ticks are slow (>100ms):

1. Check database query performance
2. Reduce number of updates per tick
3. Use indexes on frequently queried fields
4. Consider caching frequently accessed data

### High Memory Usage

If memory usage is high:

1. Check for memory leaks (use heap snapshots)
2. Clear unused data structures
3. Limit state changes history
4. Use object pooling for frequently created objects

### Broadcast Lag

If players experience lag:

1. Reduce broadcast frequency
2. Send smaller payloads (delta only)
3. Use binary protocols (MessagePack)
4. Compress large payloads
