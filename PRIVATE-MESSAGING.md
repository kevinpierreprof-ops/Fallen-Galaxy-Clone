# Private Messaging System Documentation

## Overview

The Private Messaging System provides secure, real-time private communication between players with features including conversation threading, unread counters, message history with pagination, search functionality, and user blocking.

## Security

### Authentication

- All endpoints require valid JWT token
- Socket.IO connections authenticated via token

### Authorization

- Users can only read their own messages
- Users can only mark their received messages as read
- Users can only delete messages for themselves (soft delete)

### Validation

- Message content max length: 5000 characters
- Empty messages rejected
- Cannot send to blocked users
- Cannot send to self

### Rate Limiting

Recommended rate limits:
- Send message: 10 per minute
- Mark as read: 100 per minute
- Search: 20 per minute
