## Movement Queue

### Queue Structure

```typescript
interface MovementQueueEntry {
  shipId: string;
  movement: {
    origin: string;
    destination: string;
    startTime: number;
    arrivalTime: number;
    distance: number;
    travelTime: number;
  };
  ownerId: string;
}
```

### Queue Operations

**Add to Queue:**
```typescript
movementManager.moveShip(shipId, destinationPlanetId);
// Automatically added to queue
```

**Remove from Queue:**
```typescript
movementManager.cancelMovement(shipId);
// Removed from queue
```

**Get Active Movements:**
```typescript
const movements = movementManager.getActiveMovements();
const playerMovements = movementManager.getPlayerMovements('player-1');
const toPlanet = movementManager.getMovementsToPlanet('planet-123');
```

### Auto-Completion

The queue automatically processes movements every second:

```typescript
// Update loop (runs every 1 second)
setInterval(() => {
  processMovements(); // Check arrivals
  // Ships that arrived are removed from queue
  // Arrival events are broadcast
}, 1000);
```

## Collision Detection

### Rules

1. **Neutral planets** - No collision, free to move
2. **Own planets** - No collision
3. **Enemy planets** - Collision detected, requires combat flag

### Checking Collision

```typescript
const collision = movementManager.checkCollision(planetId, playerId);

if (collision) {
  console.log('Planet occupied by enemy');
  // Show combat confirmation dialog
}
```

### Moving with Combat

```typescript
// Without combat flag - fails if enemy planet
movementManager.moveShip(shipId, planetId, false);

// With combat flag - initiates combat
movementManager.moveShip(shipId, planetId, true);
```
