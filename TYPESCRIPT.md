This project uses comprehensive TypeScript configurations with strict type checking, path aliases, and optimizations for both development and production environments.

## Type Definitions

### Backend Type Files

#### `backend/src/types/env.d.ts`
Environment variable type definitions:
```typescript
interface ProcessEnv {
  NODE_ENV: 'development' | 'production' | 'test';
  PORT: string;
  MONGODB_URI: string;
  // ... etc

```

#### `backend/src/types/socket.d.ts`
Socket.io event type definitions with full type safety:
```typescript
interface ServerToClientEvents {
  'game:update': (state: GameState) => void;
  // ... etc
}
```

#### `frontend/src/types/vite-env.d.ts`
Vite environment variables and asset imports:
```typescript
interface ImportMetaEnv {

```

Global browser type augmentations.

## Troubleshooting

### Path Alias Not Resolving

1. Restart TypeScript server: `Cmd/Ctrl + Shift + P` ÃƒÆ’Ã†â€™Ãƒâ€šÃ‚Â¢ÃƒÆ’Ã‚Â¢ÃƒÂ¢Ã¢â‚¬Å¡Ã‚Â¬Ãƒâ€šÃ‚Â ÃƒÆ’Ã‚Â¢ÃƒÂ¢Ã¢â‚¬Å¡Ã‚Â¬ÃƒÂ¢Ã¢â‚¬Å¾Ã‚Â¢ "Restart TS Server"
2. Check `tsconfig.json` paths match your file structure
3. Verify `baseUrl` is set correctly

### Type Errors in Shared Files

Ensure both frontend and backend include the shared folder:
```json
{
  "include": ["src", "../shared"]
}
```

### Module Resolution Issues

Backend uses `tsconfig-paths` for runtime:
```bash
ts-node -r tsconfig-paths/register src/server.ts
```

Frontend uses Vite config:
```typescript
resolve: {
  alias: {
    '@': path.resolve(__dirname, './src'),
    '@shared': path.resolve(__dirname, '../shared')
  }
}
```

## Migration from Existing Code

If migrating from JavaScript:

1. Start with `strict: false`
2. Enable one strict option at a time
3. Fix errors incrementally
4. Enable full strict mode when ready
