# Galaxy Canvas Component Documentation

## Overview

The `GalaxyCanvas` component is a high-performance, interactive HTML5 Canvas-based visualization for rendering a space strategy game's galaxy map. It supports pan, zoom, planet selection, ship movement visualization, and a minimap.

## Performance Optimizations

### Frustum Culling

Only render objects within the visible viewport:

```typescript
const visiblePlanets = FrustumCuller.getVisibleObjects(
  planets,
  camera.x,
  camera.y,
  viewWidth,
  viewHeight,
  zoom
);
```

### Spatial Hashing

Efficient collision detection using spatial hash grid:

```typescript
const spatialGrid = new SpatialHashGrid(100);

// Add objects
planets.forEach(p => spatialGrid.add(p.id, p.position.x, p.position.y));

// Find nearby objects
const nearby = spatialGrid.getNearby(x, y, radius);
```

### Object Pooling

Reduce garbage collection with object reuse:

```typescript
const pool = new ObjectPool(
  () => ({ x: 0, y: 0 }),
  (obj) => { obj.x = 0; obj.y = 0; },
  100
);

const point = pool.get();
// Use point...
pool.release(point);
```

### Double Buffering

Prevent flickering with offscreen rendering:

```typescript
const buffer = new DoubleBuffer(width, height);

// Draw to offscreen canvas
const offscreenCtx = buffer.getContext();
// ... render to offscreenCtx

// Copy to main canvas
buffer.copyTo(mainCtx);
```

### Performance Monitoring

Track FPS and frame time:

```typescript
const monitor = new PerformanceMonitor();

function animate() {
  monitor.update();
  
  console.log(`FPS: ${monitor.getFPS()}`);
  console.log(`Frame time: ${monitor.getFrameTime()}ms`);
  
  requestAnimationFrame(animate);
}
```

Manage canvas state and rendering:

```typescript
const {
  canvasRef,
  camera,
  fps,
  frameTime,
  setCamera,
  centerOnPosition,
  worldToScreen,

  width: 1200,

  initialZoom: 1,

  maxZoom: 3,
  enablePerformanceMonitoring: true

```

### useCanvasMouse

Handle mouse interactions:

```typescript
const { isDragging, mousePos } = useCanvasMouse({
  canvasRef,
  onDrag: (dx, dy) => pan(dx, dy),
  onClick: (x, y) => handleClick(x, y),
  onZoom: (delta, x, y) => zoom(delta, x, y)
});
```

    setSelectedPlanetId(id);

  return (
    <div>
      <GalaxyCanvas
        planets={planets}
        ships={ships}
        currentPlayerId="player-1"
        playerAlliance={['ally-1']}
        selectedPlanetId={selectedPlanetId}
        onPlanetSelect={handlePlanetSelect}
        width={1600}
        height={900}
      />
      
      {selectedPlanetId && (
        <PlanetDetailsPanel planetId={selectedPlanetId} />
      )}
    </div>
  );
}

## Troubleshooting

### Low FPS

1. **Check planet count** - Too many planets?
2. **Enable frustum culling** - Only render visible objects
3. **Reduce zoom** - Lower zoom = less detail needed
4. **Monitor performance** - Use `PerformanceMonitor`

### Choppy Animation

1. **Check frame time** - Should be ~16ms
2. **Use double buffering** - Prevents flickering
3. **Optimize render loop** - Remove unnecessary calculations

### Memory Leaks

1. **Cancel animation frames** - Clean up in useEffect
2. **Clear event listeners** - Remove on unmount
3. **Use object pooling** - Reduce garbage collection
